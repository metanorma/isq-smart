## Problem (original)

The ontology namespace `isoiec80000` is hardcoded as string literals across 3 repos, 50+ files. The prefix, URI, and derived values (qnames, slugs) are scattered rather than flowing from a single definition.

**Current namespace:**
- Prefix: `isoiec80000`
- URI: `https://w3id.org/standards/isoiec80000/ontologies/core/`

**Target namespace:**
- Prefix: `isq`
- URI: `https://w3id.org/standards/isq/ontologies/core/`

## Execution Order

1. **sdu-smart**: Rename namespace class, URI, prefix. Run specs.
2. **isq**: Update all subject URIs, types, export context. Rename spec file. Run specs.
3. **isq-smart/browser**:
   - Rename TTL files + update content
   - Rename JSON-LD context
   - Update `vite.config.ts`
   - Run `vite build` (regenerates `ontology.ts`)
   - Update Vue component strings
   - Run `rake export:all` (regenerates exports)
   - Run `vue-tsc --noEmit` + `vite build`

## Dependency Note

Since the isq gem depends on sdu-smart, and the browser imports generated data from both, the three repos must be released in order: sdu-smart → isq → isq-smart.

## Part 2 Sub-section Files

The TTL files contain part individuals like `isoiec80000:part-2-5`. These become `isq:part-2-5`. The PartRegistry `partKey: '2-5'` with `parentPart: '2'` pattern stays the same — only the RDF namespace changes.
