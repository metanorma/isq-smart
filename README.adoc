== Overview

The **ISQ Browser** is a static web application for browsing the
https://www.iso.org/standard/30669.html[ISO 80000 & IEC 80000] series of international standards
defining quantities, units of measurement, and mathematical notation.

The browser is built as a fully static SPA with no backend. All data is compiled into
JavaScript bundles at build time from YAML and Turtle sources.

== Quick start

.Prerequisites
* Node.js 22+
* Git

. Clone this repo:
+
[source,bash]
----
git clone https://github.com/metanorma/isq-smart.git
cd isq-smart
----

. Clone the three data repos into the repo root:
+
[source,bash]
----
git clone https://github.com/metanorma/iso-iec-80000.git    # YAML source data (private repo)
git clone https://github.com/unitsml/unitsdb.git             # UnitsML units/dimensions
git clone https://github.com/metanorma/sdu-smart.git         # SmartSDU TTL (ontology browser)
----
+
[NOTE]
====
`metanorma/iso-iec-80000` is a private repo — you need access to the Metanorma organization.

If you keep data repos elsewhere, set env vars to point to them:
====
+
[source,bash]
----
ISO_80000_DIR=../iso-iec-80000 UNITSDB_DIR=../unitsdb SDU_SMART_DIR=../sdu-smart npm run dev
----

. Install and run:
+
[source,bash]
----
cd browser
npm install
npm run dev          # → http://localhost:5173
----

== Architecture

The browser has **no runtime API**. Everything is compiled at build time by two Vite plugins.

=== Generated files

The Vite `yaml-data` plugin reads YAML and generates ~40 TypeScript modules into `browser/src/data/generated/`:

* `part-*.ts` — per-part entry data, math caches, bilingual text
* `meta.ts` — part metadata (titles, icons, accent colors)
* `domain-index.ts` — lightweight summaries for listing pages
* `unitsdb.ts` — UnitsML dimensional analysis database
* `physical-dimensions.ts` — dimension vectors with quantity linkage
* `ontology.ts` — parsed OWL entities from Turtle files
* `xref-map.ts` / `reverse-xref.ts` — cross-reference indexes

The `ontology-data` plugin reads Turtle from `sdu-smart/reference-docs/` and `browser/public/ontologies/`.

== Browser routing

[cols="2,3"]
|===
| Route | Page

| `/`
| Index — overview, domain cards, entry distribution chart

| `/quantities`
| Quantities domain — searchable listing of all quantity entries

| `/quantities/part-:part`
| Part listing — entries for one part (e.g. Part 4: Mechanics)

| `/quantities/part-:part/:id`
| Entry detail — single quantity with definition, units, remarks

| `/math`
| Mathematical notation domain

| `/math/part-:part/:id`
| Math entry detail

| `/units`
| Units browser (UnitsML database)

| `/units/:slug`
| Unit detail page

| `/dimensions`
| Dimensional analysis browser

| `/dimensions/:part`
| Dimension detail — quantity cross-references for a dimension

| `/documents`
| Publications listing

| `/documents/part-:part`
| Part document (scope, publisher, store link)

| `/documents/:id`
| Document detail (clauses and terms)

| `/documents/:partKey/sections`
| Document section listing

| `/documents/:partKey/sections/:sectionId`
| Section content view

| `/ontology`
| OWL ontology browser — classes, properties, SHACL shapes, SKOS concepts

| `/ontology/:slug`
| Ontology entity detail

| `/reference`
| Technical reference (URN patterns, identifier schemes)

| `/about`
| Project background, timeline, ecosystem
|===

== Deployment

The build produces a fully static site in `browser/dist/`.

[IMPORTANT]
====
Configure the host to redirect all paths to `index.html` for client-side routing.
The app uses `createWebHistory()` (HTML5 history mode).
====

=== GitHub Pages

Push `browser/dist/` contents to the `gh-pages` branch.

== CI

The GHA workflow (`.github/workflows/build.yml`) runs on push/PR to `main`:

. Checkouts this repo
. Checkouts the three data repos as workspace subdirectories:
  `iso-iec-80000`, `unitsdb`, `sdu-smart`
. Installs Node dependencies
. Runs `vite build` (production build)
. Uploads `dist/` as an artifact
